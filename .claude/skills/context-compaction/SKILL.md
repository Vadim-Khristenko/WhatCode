# Context Compaction (сжатие контекста)

Этот навык сворачивает накопившуюся историю в плотную структурированную сводку,
освобождая контекстное окно без потери смысла. Зеркалит рантайм-движок
`herta-core::context::ContextManager`, который делает то же автоматически внутри
ассистента.

## Когда срабатывать

Запускай навык, НЕ дожидаясь явной команды, если выполнено любое из:

- Оценка занятости контекста ≥ ~75% (порог по умолчанию у движка — 80%).
- Появились системные предупреждения о приближении к лимиту окна.
- История диалога стала длинной и повторяющейся; ранние детали уже неактуальны.
- Предстоит объёмная многошаговая работа, и нужно освободить место заранее.
- Пользователь просит продолжить «с того же места» в новой/длинной сессии.

## Что сохранять (НЕ отбрасывать)

1. **Стабильные факты**: окружение, версии, пути, имена, ключевые ограничения.
2. **Принятые решения и их причины**: архитектурные выборы, договорённости.
3. **Состояние задач**: что сделано, что в работе, что осталось (чек-лист).
4. **Контекст кода**: затронутые файлы/функции, инварианты, нерешённые баги.
5. **Открытые вопросы** к пользователю и ожидаемые ответы.

## Что отбрасывать

- Болтовню, повторы, промежуточные черновики, отменённые подходы.
- Полные дампы файлов и логов — заменяй ссылкой (`path:line`) и одной строкой суть.
- Эмоциональный шум и вежливые формулы.

## Алгоритм

1. Определи закреплённый префикс (системные инструкции, персона) — его НЕ трогай.
2. Сохрани дословно последние 4–6 реплик — это «живой» хвост диалога.
3. Всё между префиксом и хвостом сожми в маркированную сводку по разделам:
   `Факты`, `Решения`, `Состояние задач`, `Код/файлы`, `Открытые вопросы`.
4. Пиши плотно, в третьем лице, по делу, без воды и без выдумок.
5. Замени свёрнутый фрагмент одним блоком-сводкой и продолжай работу.

## Формат сводки

```
## Сводка контекста (сжато)
### Факты
- ...
### Решения
- ... (почему)
### Состояние задач
- [x] выполнено ...
- [ ] осталось ...
### Код / файлы
- path/to/file.rs:42 — суть
### Открытые вопросы
- ...
```

## Жёсткие правила

- НИКОГДА не выдумывай факты, которых не было в истории.
- НИКОГДА не выбрасывай незавершённые задачи или открытые вопросы.
- Сводка должна быть строго короче исходного фрагмента, иначе сжатие бессмысленно.
- После сжатия сразу продолжай прерванную работу с сохранённого хвоста.
