# Архитектура (Rust)

Воркспейс из шести crate-ов. Граф зависимостей — строго направленный, без циклов:

```
herta-core  ◄── herta-llm ◄─┐
     ▲           ▲          │
     │           │          │
herta-tools ─────┘          │
     ▲                      │
herta-agent ◄───────────────┘
     ▲
herta-tui ◄── (core, llm, tools, agent)
     ▲
herta-cli  ── (всё)
```

## Принципы

- **Никаких `panic!` в библиотеках.** Только `Result` и единый enum
  `HertaError` (`thiserror`). TUI восстанавливает терминал при любом исходе.
- **Данные отделены от поведения.** В TUI структурная конфигурация (`Theme`)
  отделена от изменчивого `AppState`, чтобы исключить конфликты заимствования с
  `&mut Frame`.
- **Детерминированный рантайм.** Один `tokio::select!`-селектор сводит события
  терминала, ответы модели и поток саб-агентов; тяжёлая работа — в отдельных
  тасках, главный поток только читает каналы и рендерит.
- **Жёсткая раскладка.** Только `Constraint::Length`/`Constraint::Min`, без
  `Percentage`, плюс ручная оценка переноса строк для безопасной прокрутки — без
  выхода за границы сетки.
- **Стиль через ratatui.** Все цвета — явные `Color::Rgb`, никаких сырых
  ANSI-литералов.

## herta-core

- `error` — `HertaError` + `Result`.
- `message` — `Role`, `Message`, эвристическая оценка токенов.
- `config` — `AppConfig` и секции, загрузка из `.env`/окружения.
- `persona` — персона Герты: ядро, догмы, правила, few-shot, **канонический лор
  HSR**, выбор полного/компактного промпта, помощники идентичности/починки.
- `memory` — кратковременная история диалога (JSON).
- `long_memory` — долговременные факты с дедупликацией.
- `context` — `ContextManager`: детерминированное решение об автосжатии,
  построение запроса к суммаризатору, применение сводки.
- `tool` — примитивы function-calling (`ToolSpec`, `ToolCall`, `ToolResult`).

## herta-llm

Async-трейт `ChatClient` (объектно-безопасный). Реализации:

- `ollama` — `/api/chat`, нативные tools, прогрев.
- `openai_compat` — `/chat/completions` для Cerebras и DeepSeek.
- `google` — `generateContent` (Gemini/Gemma) с резервной моделью.
- `anthropic` — `/v1/messages` (Claude); raw HTTP, native tool_use/tool_result,
  без `temperature` для 4.x. Модель по умолчанию `claude-opus-4-8`.

## herta-agent (дополнено)

`run_tool_loop` — провайдер-агностичный нативный tool-loop: модель вызывает
инструменты реестра, результаты подаются обратно как реплика пользователя
(без хрупкой реконструкции native tool_result), цикл ограничен числом итераций.

## herta-voice

Озвучивание ответов (TTS) через системную утилиту (`say`/`espeak-ng`/PowerShell)
без нативных аудио-зависимостей. Распознавание речи (STT) — следующая итерация.

## Навыки (`.herta`)

Собственный компактный формат с прогрессивным раскрытием: `herta-core::skill`
парсит файлы, а инструменты `list_skills`/`use_skill` дают агенту каталог и
тело по требованию. Навык `context-compaction` зеркалит рантайм-движок.

Общее: повторы с экспоненциальным backoff (`retry`), очистка `<think>`
(`sanitize_reply`), фабрика `build_client`.

## herta-tools

`ToolRegistry` хранит `dyn Tool`, отдаёт схемы модели и диспетчеризует вызовы,
блокируя деструктивные действия. Встроенные: память (`remember`/`recall`/
`forget`), `web_search` (Tavily), анализ кода (`type_check`/`lint_code`),
системные действия (`open_url`/`create_note`). Проверки в `safety`.

## herta-agent

`Supervisor` запускает саб-агентов («марионеток») в tokio-тасках с ограничением
параллелизма (семафор) и таймаутом, стримя `AgentEvent` в канал. `run_to_completion`
удобен вне TUI.

## herta-tui

- `theme` — палитра «ледяной эрудиции» (`Color::Rgb`).
- `state` — `AppState`, лента, ввод, агенты, прокрутка, индикатор контекста.
- `ui` — чистый рендер кадра по состоянию.
- `app` — async-цикл: ввод, ответы модели, события агентов, автосжатие контекста.

## herta-cli

Бинарь `herta`: TUI по умолчанию, `--text` для одноразового запроса, `doctor`
для самодиагностики.
